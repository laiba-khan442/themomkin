# The MomKin

The MomKin is a static front-end website built for a baby and maternity lifestyle brand. The project serves as a digital catalog and brand presence site, showcasing products, stories, and essential information for parents and caregivers in a clean, accessible, and mobile-friendly format.

## Overview

This website is designed to represent The MomKin brand online and provide visitors with a polished browsing experience across key pages such as Home, About, Shop, Product, Blog, and Contact. It combines branded storytelling with product discovery, helping users explore baby, kids, and maternity essentials in an intuitive and visually engaging way.

## Purpose

The primary goal of this project is to:
- Establish a professional online presence for The MomKin
- Present product collections in a catalog-style format
- Support brand storytelling through blog and content pages
- Provide a responsive and user-friendly experience across devices

## Tech Stack

This project is built using:
- HTML5
- CSS3
- JavaScript
- Bootstrap
- jQuery
- Slick Slider and supporting front-end plugins

## Project Structure

The project is organized as a static website with standalone HTML pages and shared assets:

- `index.html` – homepage
- `about.html` – brand and company information
- `shop.html` – product listing page
- `product.html` – product detail page
- `blog.html` / `blogs.html` – blog content and listings
- `contact.html` – contact page
- `assets/` – CSS, JavaScript, images, fonts, and media
- `data/` – supporting project data files

## Features

Key features of the website include:
- Responsive multi-page layout
- Brand-focused homepage hero sections
- Product catalog and promotional sections
- Blog and storytelling content
- Mobile navigation support
- Image sliders and interactive UI components
- Social media integration and brand assets

## Development Notes

This is a static website project, so it can be run locally by opening the HTML files in a browser or by serving the root directory through a local development server.

For best results during development, use a lightweight local server such as:
- VS Code Live Server
- `python -m http.server`
- any static file server of your choice
